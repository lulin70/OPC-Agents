# OPC-Agents Web界面增强计划

## [x] 任务1：实现总裁办主动决策机制 - 天气查询
- **优先级**：P0
- **依赖**：None
- **描述**：
  - 修改总裁办对话功能，当用户询问天气时，总裁办应主动决策并派发任务
  - 实现任务分派给市场部门，让其添加搜索引擎功能或寻找相应的MCP/CLI
  - 跟踪任务进度并将结果反馈给用户
- **成功标准**：
  - 当用户询问天气时，总裁办不再直接拒绝，而是创建任务并分派给相关部门
  - 部门接收任务并开始处理
  - 最终向用户提供天气信息或解决方案
- **测试要求**：
  - `programmatic` TR-1.1: 用户输入"今天上海天气如何"时，系统应创建天气查询任务并分派给市场部门
  - `human-judgement` TR-1.2: 对话流程应清晰展示总裁办的决策过程和任务分派过程
- **注意**：需要确保任务分派和跟踪的流程清晰可见

## [x] 任务2：实现总裁办主动决策机制 - 大模型连接问题

- **依赖**：None
- **描述**：

- **注意**：需要确保错误处理和任务分派的流程清晰可见

## [x] 任务3：实现互动可视化功能
- **优先级**：P1
- **依赖**：任务1、任务2
- **描述**：
  - 在Web页面下方添加部门列表
  - 实现互动箭头从发起方指向接收方
  - 点击箭头可查看任务ID
  - 实现接收方在处理进程中高亮显示
  - 实现任务完成后箭头指向更新
- **成功标准**：
  - Web页面下方显示主要部门列表
  - 当有互动时，显示箭头从发起方指向接收方
  - 点击箭头可查看任务详情
  - 正在处理任务的部门高亮显示
  - 任务完成后，箭头指向正确更新
- **测试要求**：
  - `programmatic` TR-3.1: 当任务分派时，系统应显示箭头指向目标部门
  - `programmatic` TR-3.2: 点击箭头应显示任务ID和详情
  - `human-judgement` TR-3.3: 界面应清晰展示部门间的互动流程，高亮效果明显
- **注意**：需要确保可视化效果直观且响应及时

## [x] 任务4：集成天气查询功能

- **描述**：
