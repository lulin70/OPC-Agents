# OPC-Agents 对话界面优化方案

## 目标
设计一个能够清晰展示总裁办思考过程、任务派发、等待状态、完成状态以及被调用Agent进度的对话界面。

## 参考设计
参考用户提供的WorkBuddy界面，设计一个类似的对话流程展示系统。

## 界面设计方案

### 6. 优势

1. **透明度**：用户可以清晰看到总裁办的思考过程和任务分派

3. **完整性**：从任务接收到完成的全流程展示
4. **可追溯性**：所有对话和任务历史都有记录
5. **交互性**：用户可以干预任务执行过程

## 8. 对话历史管理优化

### 8.1 智能排序策略

**优先排序规则**：
1. **最近活跃任务**：按任务的最后活动时间排序（包括最后消息、进度更新、状态变化）
2. **任务类型权重**：
   - 进行中任务 > 计划中任务 > 已完成任务
   - 高优先级任务 > 中优先级任务 > 低优先级任务
3. **任务名称可编辑**：用户可以修改任务名称，使其更具描述性

### 8.2 对话历史分组

**分组方式**：
- **今天**：今天有活动的任务
- **本周**：本周内有活动的任务
- **长期任务**：持续时间超过7天的任务
- **重复任务**：定期重复执行的任务
- **已完成**：已完成但未归档的任务
- **归档**：已完成且已归档的任务

### 8.3 任务卡片设计

**任务卡片信息**：
```
┌─────────────────────────────────────────┐
│ [进行中] 产品发布计划                  │
│ ⏰ 最后活跃: 10分钟前                 │
│ 👥 参与: 市场部、产品部、项目管理部    │
│ 📊 进度: ████████░░ 80%             │
│ 💬 32条消息                          │
└─────────────────────────────────────────┘
```

### 8.4 重复任务管理

**重复任务处理**：
- 每条重复任务创建独立的对话实例
- 保留历史执行记录
- 显示下次执行时间
- 提供"完成本次"和"跳过"选项

### 8.5 长期任务管理

**长期任务处理**：
- 自动分段：将长期任务拆分为阶段性子任务
- 里程碑标记：在对话中显示关键里程碑
- 进度趋势：显示任务进度的历史趋势
- 定期提醒：根据任务重要性设置提醒

### 8.6 界面实现

**前端展示**：
- 左侧导航栏：按智能分组显示任务
- 搜索功能：支持按任务名称、关键词、部门、代理搜索
- 筛选功能：按任务状态、优先级、部门筛选
- 排序选项：提供多种排序方式（最后活跃、创建时间、优先级）

**交互功能**：
- 任务名称编辑：点击任务标题直接修改
- 任务归档：将已完成任务移至归档
- 任务置顶：将重要任务固定在顶部
- 任务标记：为任务添加标签（如"重要"、"紧急"）

## 9. 结论

通过这套设计，用户可以在一个统一的对话界面中：
- 看到总裁办的思考过程
- 了解任务的分派情况
- 实时监控任务的执行进度
- 查看任务的完成结果
- 追溯整个流程的历史记录

同时，通过智能的对话历史管理：
- 优先显示最近活跃的任务
- 按任务性质和状态智能分组
- 支持长期任务分段和重复任务处理
- 允许用户灵活修改任务名称
- 提供强大的搜索和筛选功能

这种设计不仅提高了系统的透明度和用户体验，也让用户能够更好地理解和参与到任务执行过程中，同时有效地管理长期和重复任务。
