# TRAE Agency-Agents 集成指南

## 系统概述

Agency-Agents 是一个 AI 驱动的代理系统，集成了 180 个来自官方项目的专业代理，覆盖多个部门和领域。通过 TRAE 技能集成，您可以直接在 TRAE 界面中使用这些代理来完成各种任务。

## 安装和配置

### 1. 确保项目文件存在

项目应该位于：`/Users/lin/Documents/trae_projects/agency-agents`

### 2. 技能文件配置

由于权限限制，我们需要手动配置 TRAE 技能：

1. 打开 TRAE 应用
2. 进入「技能」管理界面
3. 创建一个新技能，名称为「trae-agency」
4. 将 `agency_skill.py` 的内容复制到技能实现中
5. 保存技能配置

## 使用方法

在 TRAE 对话界面中，您可以使用以下命令：

### 1. 查看所有部门

```
TRAEE, 查看所有部门
```

此命令将显示所有可用的部门及其代理数量。

### 2. 查看特定部门的代理

```
TRAEE, 查看部门 engineering
```

此命令将显示指定部门的所有代理及其描述。

### 3. 分配任务给代理

```
TRAEE, 安排任务 engineering 设计一个高性能的数据库架构 --agent "Database Optimizer"
```

此命令将把任务分配给指定部门的指定代理。

### 4. 创建项目

```
TRAEE, 创建项目 "电商平台开发" "design:设计电商网站UI" "engineering:实现用户认证系统"
```

此命令将创建一个包含多个任务的项目，并将任务分配给相应部门的代理。

## 示例用法

### 示例 1：创建一个网站开发项目

```
TRAEE, 创建项目 "企业官网开发" "design:设计响应式网站UI" "engineering:实现前端页面" "engineering:实现后端API" "marketing:制定网站推广策略"
```

### 示例 2：分配具体任务

```
TRAEE, 安排任务 design 设计一个现代简约的登录页面 --agent "UI Designer"
```

### 示例 3：查看部门代理

```
TRAEE, 查看部门 marketing
```

## 故障排除

### 常见问题

1. **命令不生效**：确保命令格式正确，特别是空格和参数顺序。
2. **部门或代理不存在**：使用「查看所有部门」命令确认部门名称正确。
3. **任务分配失败**：检查部门是否存在，以及代理名称是否正确。

### 错误信息

- `Department not found`：部门名称错误，请检查部门列表。
- `No agent found in department`：部门中没有找到指定的代理，请检查代理名称。

## 性能优化

- 系统会自动加载官方代理数据，首次使用可能需要几秒钟时间。
- 对于大型项目，建议分批次创建任务，以避免系统负载过高。
- 代理响应使用本地模拟模式，实际部署时可以连接到 TRAE API 获取真实响应。
